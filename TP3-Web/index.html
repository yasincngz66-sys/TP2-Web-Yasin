EXERCICE 1 : Glossaire des balises HTML

Balise Explication
<!DOCTYPE html> Indique au navigateur que la page utilise la dernière version de HTML (HTML5).
<html> La racine du document : elle englobe tout le code de la page.
<head> La tête du fichier : zone de configuration invisible pour le visiteur (réglages, liens).
<meta> Fournit des réglages techniques (comme les accents ou l'adaptation aux mobiles).
<title> Le texte qui s'affiche tout en haut dans l'onglet du navigateur.
<link> Fait le pont vers un fichier externe, généralement pour le design (CSS).
<body> Le corps de la page : tout ce qui est affiché à l'écran pour l'utilisateur.
<header> L'introduction d'une page ou d'une section.
<div> Une boîte vide utilisée pour regrouper des éléments afin de les mettre en page.
<article> Bloc de contenu autonome qui a du sens même lu seul .
<h1> Le titre le plus important de la page.
<p> Balise standard pour écrire un paragraphe de texte.
<ul> Crée une liste d'éléments marqués par des puces.
<footer> Le pied de page.
<script> Permet d'ajouter de l'interactivité ou des animations via le langage JavaScript.

EXERCICE 2 : Reproduction du texte SQL

1. Le langage SQL

1.1. Définition

Le langage SQL (Structured Query Language ou Langage de requêtes structuré) est un langage comprenant à lui seul l'ensemble des ordres nécessaires à la création et à la gestion d'une base de données relationnelle.

Il a été créé en 1986 et est en fait l'aboutissement de plusieurs langages (Sequel en 1977, System/R, ...) créés depuis l'apparition du modèle relationnel en 1970 (le modèle relationnel a été inventé par Mr Codd, un mathématicien de chez IBM).

L'utilisation du langage SQL suppose obligatoirement que les données aient été définies selon le modèle relationnel au sein d'un SGBDR (Oracle, SQLServer, mySQL, PostgreSQL...).

1.2. Les différentes normes utilisées

Il existe aujourd'hui plusieurs normes relatives à l'utilisation du langage SQL.

La différence entre ces deux normes se situe principalement dans l'écriture des jointures et dans l'ajout de fonctionnalités :

  • La norme SQL 89 qui est encore très utilisée actuellement.
  • La norme SQL 92.
  • Aujourd'hui SQL 2011.

Bien qu'il existe des normes il peut y avoir des différences entre SGBDR dans la syntaxe.

Ainsi on peut lire sur le site de PostgreSQL :

"Le développement de PostgreSQL respecte le standard en vigueur, tant que celui-ci ne s'oppose pas aux fonctionnalités traditionnelles ou au bon sens..."
"SQL-92 définit trois niveaux de conformité: basique (ENTRY), intermédiaire (INTERMEDIATE) et complète (FULL). La majorité des systèmes de gestion de bases de données se prétendaient compatibles au standard SQL dès lors qu'ils se conformaient au niveau Entry"
